# NNUE-mlx

NNUE (Efficiently Updatable Neural Network) board game AI, trained on Apple Silicon via [MLX](https://github.com/ml-explore/mlx) with unified memory.

Includes four playable games with Pygame GUI, alpha-beta search engine, and a complete self-play training pipeline.

## Games

| Game | Board | Notes |
|------|-------|-------|
| Chess | 8x8 | Full rules (castling, en passant, promotion) |
| Los Alamos Chess | 6x6 | No bishops, no castling, pawns promote to queen only |
| Shogi | 9x9 | Piece drops, promotions, nifu/uchifuzume rules |
| Mini Shogi | 5x5 | Simplified shogi with 6 piece types per side |

## Requirements

- macOS with Apple Silicon (M1/M2/M3/M4)
- [uv](https://docs.astral.sh/uv/) package manager

## Quick Start

```bash
# Install dependencies and build the accelerated inference engine
uv sync
make build-accel

# Play a game (opens Pygame window with game selection menu)
uv run python scripts/play.py
```

## Training

### Iterative Self-Play Pipeline

The recommended approach is the iterative pipeline: bootstrap with a rule-based evaluator, then loop through self-play data generation, training, and evaluation.

```bash
# 1. Build the NEON SIMD + Accelerate inference engine
make build-accel

# 2. Run the full iterative pipeline (bootstrap → generate → train → evaluate → repeat)
uv run python scripts/pipeline.py --game minichess --iterations 6

# 3. Play against the trained AI
uv run python scripts/play.py --game minichess --mode human-vs-ai --model models/<timestamp>/model.npz
```

Key pipeline features:

- **Rule-Based bootstrap**: initial data generated by alpha-beta search with PeSTO piece-square tables, pawn-structure and king-safety terms (not random play). Chess and shogi rule evaluators have C-native counterparts kept in strict parity via regression tests.
- **Cosine-annealed LR** with lambda schedule (1.0 → 0.75) and gradient clipping
- **Output bucketing + optional WDL head + feature factorization** (SOTA NNUE training stack)
- **Early stopping** on validation loss with best-model checkpointing
- **Warm-start** from previous iteration's model weights
- **Opening book + adjudication** during self-play for diverse, decisive games

### Standalone Training

For manual single-shot training:

```bash
uv run python scripts/train.py --game minichess --data data/minichess_*.bin --epochs 50
```

To evaluate a trained model against the material-counting baseline:

```bash
uv run python scripts/evaluate.py --game minichess --model1 models/minichess_final.npz --model2 material --games 100
```

## Accelerated Inference

The C extension accelerates the search engine using ARM NEON SIMD intrinsics and Apple's Accelerate framework (`cblas_sgemv` / AMX coprocessor). Once built via `make build-accel`, it is used automatically.

If the extension is not built, the engine falls back to a pure-numpy implementation.

| Component | Without Extension | With Extension | Speedup |
|-----------|-------------------|----------------|---------|
| push/pop | ~2.0 us | 0.07 us | 29x |
| update | ~2.0 us | 0.08 us | 26x |
| evaluate | ~8.0 us | 1.3 us | 6x |
| **Per search node** | **12.5 us** | **1.5 us** | **8.4x** |

## Architecture

```
HalfKP Features --> Embedding(256) + ClippedReLU
                    x2 perspectives (white/black), shared weights
                          |
                    Concat (512)
                          |
                    Linear(128) + SCReLU
                          |
                    Linear(32)  + SCReLU
                          |
                    Linear(num_buckets)   --> eval score (piece-count bucketed)
                    Linear(num_buckets)   --> optional WDL logits (dual head)
```

- **Activation**: ClippedReLU for the feature transformer (incremental accumulator compatible); SCReLU (`clamp(x,0,1)²`) for hidden layers (~50% effective capacity increase)
- **Output bucketing**: 8 output heads selected by total piece count, so the network can learn phase-specific evaluations (opening / middlegame / endgame) without having to encode phase into the feature space.
- **Dual WDL head** (optional): a second linear head outputs win/draw/loss logits alongside the scalar eval, providing richer training signal when game outcomes are available.
- **Feature factorization** (optional): virtual parent features sum into the main HalfKP embeddings during training for faster convergence; they're folded away at inference so the serving model is unchanged.
- **Int8 quantization**: all hidden layers (L1/L2/output/WDL) support full-int8 inference for the C accel path.
- **Training**: MLX on Apple Silicon (unified memory, zero-copy CPU/GPU, JIT compilation). Cosine-annealed learning rate, lambda schedule (eval/result mix 1.0 → 0.75), gradient clipping, horizontal mirror augmentation, and early stopping. Loss uses `|p - target|^2.6` with sigmoid win-probability scaling.
- **Search**: Alpha-beta with quiescence search, null move pruning, LMR, futility pruning, PVS, aspiration windows, and check extensions. Shipped in both a single-threaded Python path and a C-native multi-threaded Lazy SMP path. Incremental accumulator on CPU, accelerated with NEON SIMD + Apple Accelerate framework.
- **GUI**: Pygame front-end with live analysis (eval summary, +/-6 pawn vertical eval bar, score history chart), board flip (F), and Play-As side selection for human-vs-AI. Shogi and minishogi render Japanese kanji with promoted-piece highlighting.
